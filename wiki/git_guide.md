# Git 버전 관리 가이드 🐙

프로젝트의 변경 사항을 저장(GitHub 업로드)하거나, 문제가 생겼을 때 이전 시점으로 되돌리는 방법을 설명합니다.

---

## 1. 변경 사항 저장하고 업로드하기 (Save & Upload)

작업한 코드를 저장하고 GitHub에 올리는 3단계 과정입니다.

### 1단계: 변경된 파일 확인 및 담기
현재 수정된 파일들을 '바구니(Staging Area)'에 담습니다.
```bash
git add .
```
*(점 `.`은 모든 변경 파일을 의미합니다)*

### 2단계: 설명과 함께 저장하기 (Commit)
바구니에 담긴 파일들을 확정하여 저장합니다. 이때 변경 내용을 설명하는 메시지를 적습니다.
```bash
git commit -m "작업한 내용 요약 (예: 검색 기능 추가)"
```

### 3단계: GitHub에 올리기 (Push)
내 컴퓨터에 저장된 내용을 GitHub(원격 저장소)로 보냅니다.
```bash
git push
```
*(처음 한 번만 `git push -u origin main`을 입력하면, 이후에는 `git push`만 입력해도 됩니다)*

---

## 2. 이전 버전으로 되돌리기 (Undo & Revert)

작업 중 실수를 했거나, 특정 시점으로 돌아가고 싶을 때 사용합니다.

### 케이스 A: 방금 수정한 내용 취소하기 (커밋 전)
아직 `git commit`을 하지 않았는데, 방금 작성한 코드를 모두 지우고 마지막 저장 상태로 돌아가고 싶을 때:
```bash
git checkout .
```
*(주의: 저장하지 않은 코드는 모두 사라집니다)*

### 케이스 B: 특정 작업만 취소하기 (Revert)
이미 커밋을 해서 GitHub에도 올라갔지만, 특정 시점의 작업 내용을 취소하고 싶을 때 (안전한 방법):
1. **히스토리 확인**: `git log`를 입력하여 취소하고 싶은 커밋의 ID(노란색 문자열)를 복사합니다. (`q`키를 눌러 빠져나옵니다)
2. **취소 커밋 생성**:
```bash
git revert <커밋ID>
```
*예: `git revert a1b2c3d`*
(이 방법은 기존 기록을 지우지 않고, "취소했다"는 새로운 기록을 남기는 안전한 방법입니다)

### 케이스 C: 과거 시점으로 완전히 되돌리기 (Reset)
**주의**: 특정 시점 이후의 모든 기록을 지워버리고 강제로 돌아갑니다. 협업 중에는 권장하지 않습니다.
```bash
git reset --hard <돌아가고싶은_커밋ID>
```

---

## 3. 현재 상태 확인하기
어떤 파일이 수정되었는지, 어떤 브랜치에 있는지 확인할 때 가장 자주 사용하는 명령어입니다.
```bash
git status
```
